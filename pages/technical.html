🔄 データフロー・処理順序

1
音声入力キャプチャ

Web Speech APIでリアルタイム音声認識、文字起こし結果をTranscriptに蓄積

navigator.mediaDevices.getUserMedia() → webkitSpeechRecognition
2
コマンド解析・発話権制御

「どうぞ」「文字消して」等の音声コマンドを解析、AIへの発話権を制御

3
AI応答生成

フェーズ別プロンプトでGPT-4o-miniに送信、キャラクター別応答を生成

buildConversationMessages() → fetch('/v1/chat/completions')
4
音声合成・再生

キャラクター設定に基づいてTTS音声合成、Blobで音声再生

ttsTextToAudioBlob() → new Audio(audioUrl) → audio.play()
5
知見抽出・蓄積

長文回答から価値ある部分を自動抽出、構造化してAppStateに保存

DEEPDIVE_SUMMARY(text) → extractedKnowledge.push()
6
レポート生成・エクスポート

Markdownフォーマットでセッションレポート生成、ダウンロード提供

generateSessionMarkdown() → Blob() → URL.createObjectURL()

🔒 セキュリティ・プライバシー仕様

🔐 APIキー暗号化
XOR暗号化でユーザー設定パスワードベースの暗号化。LocalStorageに保存時は平文でない暗号化データのみ。復号にはパスワードが必須で、ブルートフォース攻撃耐性あり。
🌐 通信セキュリティ
OpenAI APIとの通信は全てHTTPS。APIキーはAuthorizationヘッダーで送信。リクエスト内容はJSON形式で構造化。レスポンスデータは一時的にメモリ内のみ保持。
💾 データ保護
ブラウザのLocalStorage APIを使用せず、メモリ内状態管理を優先。セッション終了時にセンシティブデータは自動クリア。永続化は暗号化されたAPIキーのみ。
🎤 音声データ保護
音声認識はブラウザのWeb Speech API使用。音声データはローカル処理。TTSで生成された音声Blobはメモリ内のみ、再生後即座に削除。長期保存なし。
🛡️ 入力値検証
APIキー形式（sk-で始まる）の検証。ファイルサイズ・形式の制限。XSS対策でHTMLエスケープ処理。CSP (Content Security Policy) 実装済み。
👤 プライバシー設計
個人特定情報は収集せず。会話内容はセッション中のみ保持。GDPR準拠設計。ユーザーの明示的同意なしにデータを外部送信しない。オプトアウト機能あり。
